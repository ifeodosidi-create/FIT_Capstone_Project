Требования к проекту

1. Проект нужно разделить на логические части. Имена файлов можно менять, главное — чтобы было понятно, за что каждый файл отвечает. Логику не реализуем в одном файле.

Пример:

· main.py # главный файл, с которого начинается программа

· models.py # описание данных (операции, категории и т.п.)

· storage.py # сохранение и загрузка данных

· gui.py # интерфейс программы

· analysis.py # анализ и графики

· utils.py # вспомогательные функции

· data/ # файлы с данными (если используется хранение в файлах)

2. В проекте обязательно нужно использовать классы. Классы должны описывать основные сущности приложения: бронирование; отмена брони. Можно (и даже нужно) применять принципы ООП

3. Работа с файлами и сохранение данных. Программа должна запоминать данные между запусками. Допускаются варианты:

· хранение данных в базе Postgresql.

4. Приложение должно иметь графический интерфейс. Интерфейс можно сделать с помощью Flask (web-приложение).

Элементы: формы, списки, кнопки, фильтры.

Интерфейс должен позволять в зависимости от пользователя:

клиенту

· создать бронь, выбрав все доступные параметры, после создания брони вывести ее ID и возможность сохранить детали бронирования в файл;
- расчет суммы должен учитывать кол-во гостей, ночей, завтрак входит в стоимость, обед и ужин можно выбрать для каждого гостя за отдельную плату, скидка 5% за повторную бронь в течение года, скидка 5% за бронь от 3 суток, скидки суммируются
- оплатить бронь - просто кнопка - при нажатии отображается информация "здесь должен быть эквайринг" и кнопка "подтвердить платеж - да/нет" и создание или нет платежа
- отменить бронь - если до заезда есть хотя бы 1 день. Если бронь уже была оплачена - вывести инфу - за возвратом денежных средств обращайтесь по телефону 880012345678

админу

· видеть все данные;

· запускать анализ и построение графиков.

5. Программа должна уметь анализировать финансовые данные. Для анализа можно использовать библиотеку pandas.

Примеры анализа:

· сколько доходов принесли номера по каждой категории за определенное кол-во времени;

· сколько гостей было по месяцам;
самые популярные номера - топ 5 по каждой категории и т.д.

6. Программа должна обладать возможностью визуализации данных (matplotlib; seaborn). Результаты анализа должны отображаться: в виде графиков, либо сохраняться в файл.

Примеры графиков:

· график доходов по категориям и по времени;

· круговая диаграмма платежей по категориям за выбранный период;

· столбчатая диаграмма кол-ва гостей и доходов по дням за выбранный период.

7. Регулярные выражения (re) должны применяться для:

проверки корректности вводимых данных; анализа и очистки входных строк;

8. Программа должна использовать функции, не писать весь код линейно. Все основные действия должны быть оформлены как функции:

· добавление операции;

· удаление операции;

· фильтрация по категории или дате;

· расчёт доходов.

9. Программа должна быть устойчивой к ошибкам, чтобы пользователь не мог «сломать» её случайными действиями или некорректными данными (try / except).Обязательно применять try / except при работе с файлами; при взаимодействии с базой данных; при обработке пользовательского ввода.

10. Необходимо написать простые unit-тесты с использованием unittest.

Тестами нужно покрыть:

· работу классов;

· функции анализа данных.

11. Весь код должен содержать docstrings. Создать подробный README с описанием проекта.

12. Проект должен быть размещён в публичном Git-репозитории.

В репозитории должны быть:

· файл .gitignore;

· файл README.md;

· история коммитов (хотя бы несколько).
